# 标准子代理角色规范

主代理可以并行使用多个子代理，但角色职责必须稳定，不要混用。

角色之外，子代理的启动透明度和关闭责任仍由主代理负责，详见 [subagent-lifecycle.md](subagent-lifecycle.md)。

## 1. builder

用途：

- 实现局部功能
- 修改受控写入范围内的文件
- 交付具体产物

主代理派发时至少要给：

- 任务目标
- 写入范围
- 禁止修改范围
- 前置依赖
- 验收条件

builder 不负责：

- 全局方案裁定
- 最终验收
- 宣布整体完成

## 2. explorer

用途：

- 只读探索
- 方案比对
- 风险发现
- 结构审查

主代理派发时至少要给：

- 需要回答的问题
- 审查边界
- 是否只读

explorer 不负责：

- 直接实施改动
- 代替主代理做最终方向判断

## 3. verifier

用途：

- build、start、curl、测试、运行验证
- 检查实现和预期是否一致

主代理派发时至少要给：

- 要验证的对象
- 预期结果
- 验证命令或验证路径

verifier 不负责：

- 解释结果是否合理
- 替代主代理做最终接受决定

## 角色使用原则

- 关键路径上的任务优先单独派发，不与其他职责混包
- 同一任务如果既要实现又要验证，必须拆成至少两个角色
- 同一轮中可以有多个 builder 或多个 explorer，但职责边界仍要清楚
- 只有主代理可以跨角色整合结论
