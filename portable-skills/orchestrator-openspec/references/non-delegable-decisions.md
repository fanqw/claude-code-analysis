# 不可外包决定清单

这份文档只回答一个问题：

- 哪些决定必须由主代理亲自做

## 绝不能外包的决定

- 当前轮唯一目标是什么
- 是否进入 OpenSpec
- 哪个方案被正式采纳
- 是否允许进入下一阶段
- 是否允许启动 builder
- 最终裁定是 `accepted / revise / blocked`
- 当前任务是否已到可收口状态
- 当前冲突应如何解决

## 关键路径升级规则

满足任一情况时，主代理应把当前问题视为关键路径处理：

- 当前决定直接影响是否能进入下一阶段
- 当前结果直接影响是否能启动实现
- 当前结果直接影响是否可以收口
- 当前验证结果存在争议
- 当前问题一旦判断错误，会导致后续多轮返工

关键路径上的默认动作：

- 优先使用更稳妥模型
- 需要时补独立验证
- 不依赖单个子代理自述
- 主代理保留亲自复核权
