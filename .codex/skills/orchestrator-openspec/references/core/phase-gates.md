# 六阶段门禁

这份文档默认服务于中大型、需要明显编排的任务。

如果当前任务可能属于轻量、单 Agent 或低自主度任务，先结合：

- [../execution/lightweight-task-boundaries.md](../execution/lightweight-task-boundaries.md)

理解“agentic 程度”的快速分级：

- 低：主代理可直接完成，只需少量探索或单次实现
- 中：需要多步推进、阶段切换或独立验证，但不一定要并行子代理
- 高：需要规格冻结、多角色协作、多轮验收或明显的长任务治理

通常只有中到高的任务，才需要完整六阶段门禁。

## 1. Intent Lock

退出条件：

- 目标可用一句话描述
- 成功标准明确
- 范围和约束明确
- 已判断是否进入 OpenSpec
- 已判断当前任务属于低、中还是高 agentic 程度

用户介入：

- 通常在本阶段确认一次目标、范围和成功标准

## 2. Solution Exploration

退出条件：

- 备选方案已经探索
- 主代理已完成方向裁定
- 若属于中大型任务，已进入 OpenSpec change 流程
- 已判断后续是否需要多角色并行或独立 verifier

用户介入：

- 只有存在明显方案分叉时才询问用户

## 3. Decision & Brief Freeze

退出条件：

- 当前规格已冻结到可执行状态
- 可从 OpenSpec 工件中提炼出任务包
- 主代理已明确执行顺序和并行关系
- 当前规范记忆入口清楚

用户介入：

- 通常只确认大方向是否接受，不逐项确认任务拆分细节

## 4. Parallel Execution

退出条件：

- 已分配任务都有结果
- 结果状态明确为已完成、需返工或阻塞
- 当前执行记忆已能支撑下一轮续接

用户介入：

- 一般不频繁打断用户，除非出现范围变化、方向变化或关键风险升级

## 5. Acceptance & Verification

退出条件：

- 每个任务都被主代理裁定
- 关键能力经过独立验证
- 如本轮暴露新错误模式，已判断是否进入知识闭环

用户介入：

- 只在结果有争议，或质量、范围、时间取舍不明显时请用户拍板

## 6. Archive & Reporting

退出条件：

- 本轮最小证据链已记录
- 轮次总结已完成
- 如本轮彻底结束，相关子代理关闭
- 规范记忆、执行记忆、经验记忆的落点都已明确

用户介入：

- 一般不需要用户决策
