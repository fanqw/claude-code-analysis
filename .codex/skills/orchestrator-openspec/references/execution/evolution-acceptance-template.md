# 演化任务验收模板

这份模板用于 skill、文档、流程资产这类“非功能代码改动”的轻量验收收口。

它不替代 [`../core/acceptance-patterns.md`](../core/acceptance-patterns.md)，只补一个更贴近演化任务的固定骨架。

## 适用场景

优先用于这些任务：

- skill 主协议调整
- reference 分层或目录重组
- prompt / script / template 接线
- 文档工程任务
- 流程资产补齐

如果当前任务包含明显运行面、接口面或用户可见行为变化，仍优先回到常规验收和独立验证。

## 验收骨架

```md
## Acceptance

- 裁定：accepted | revise | blocked
- 演化对象：
- 本轮目标：

### 结构结果

- 本轮新增 / 调整了什么：
- 是否保持目录或协议一致：
- 是否存在悬空引用或重复入口：

### 验证结果

- 静态检查：
- 链接或路径检查：
- 脚本语法检查：
- 是否需要运行面验证：

### 三类记忆落点

- 规范记忆落点：
- 执行记忆落点：
- 经验记忆落点：

### 证据

- 关键文件：
- 命令或检查：
- 当前结论依据：

### 下一步

- 是否还有未收口项：
- 是否需要知识闭环：
```

## 最小通过条件

满足全部条件时，通常可以对演化任务给出 `accepted`：

- 本轮目标可以一句话复述
- 调整后的结构没有悬空引用
- 关键脚本或模板没有明显失配
- 三类记忆落点能指出
- 下一轮接手者能直接继续，而不是重新恢复背景

## 何时不能直接 accepted

满足任一条件时，优先给 `revise` 或 `blocked`：

- 主协议与 reference 说法冲突
- 目录重组后存在悬空路径
- 新模板没有实际接入执行流
- 本轮只是加文档，没有完成最小验证
- 下一轮无法判断该从哪里继续
