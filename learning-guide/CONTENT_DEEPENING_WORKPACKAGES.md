# 内容深修并行工作包

## 目标

围绕“引导用户学习，保证源码真实性”的中心，把后续内容深修拆成可以并行推进的工作包。目标不是同时扩写所有页面，而是先稳定教程主线，再补齐证据核验层，最后做跨页一致性整合。

## 拆分原则

- 先保主线，再补证据，最后做扩展。
- 每个工作包都必须能独立验收，避免多人同时改同一段正文。
- 章节页正文和证据核验层尽量分离，减少冲突。
- 高风险任务只在证据真实性或最终整合阶段使用更强模型，其余默认用高性价比模型。

## 工作包

### A. 全站内容骨架

- 目标：统一首页、地图页、章节页、核验页的开场文案和页面职责表达。
- 范围：
  - 首页首屏导语
  - `/map` 教程树导语
  - `/analysis` 与 `/sources` 的核验定位文案
  - 各类返回路径和辅助说明
- 产出：
  - 页面开场文案清单
  - 统一术语表
  - 页面职责对照表
- 推荐模型：`gpt-5.4-mini`

### B. 阶段主线正文

- 目标：逐章压缩、重写主线正文，让每章先回答“这章要理解什么”。
- 范围：
  - `architecture -> tool-call -> prompt -> context -> session -> sandbox`
  - 后续阶段章节按相同模板补齐
- 产出：
  - 每章的一句话结论
  - 白话解释
  - 为什么学这章
  - 最小下一步
- 推荐模型：`gpt-5.4-mini`

### C. 证据核验层

- 目标：检查 analysis 原文和源码入口是否真实、准确、可核验。
- 范围：
  - 每章挂接的 analysis 片段
  - 每章挂接的源码入口
  - 证据导航标签
- 产出：
  - 证据挂接表
  - 缺失证据清单
  - 高风险章节核验报告
- 推荐模型：
  - 默认 `gpt-5.4-mini`
  - 涉及复杂源码映射或高风险章节时再切换更强模型

### D. 页面级导语与入口文案

- 目标：清理会和主线竞争的文案，把所有入口语义统一成教程站风格。
- 范围：
  - 卡片标题
  - CTA 文案
  - `details` / `summary` 标题
  - 页脚和返回路径
- 产出：
  - 页面级 CTA 清单
  - 替换前后对照表
- 推荐模型：`gpt-5.4-mini`

### E. 最终整合审校

- 目标：统一整站语气、信息层级和交叉链接，确认没有重新长回“索引站”。
- 范围：
  - 全站术语一致性
  - 顶层导航和页脚入口一致性
  - 章节页模板一致性
  - 首页、地图页、核验页之间的叙述一致性
- 产出：
  - 最终审校清单
  - 回归问题列表
  - 冻结版发布建议
- 推荐模型：更强模型优先，用于最终统一判断

## 低成本执行顺序

1. 先跑 A，锁定全站骨架和术语。
2. 并行启动 B 和 D，一个压正文，一个压入口文案。
3. B 稳定后启动 C，避免证据挂接对象频繁变化。
4. 最后由 E 做统一审校和冻结。

## 推荐并行流水线

1. 子代理 1：负责 A 或 D，偏文案与导览语义。
2. 子代理 2：负责 B，专注章节主线正文。
3. 子代理 3：负责 C，专注 evidence 挂接与真实性核验。
4. 主代理：负责 E，统一收敛并做最终实现。

## 模型策略

- 默认策略：大多数内容改写和整理任务使用 `gpt-5.4-mini`。
- 提权条件：
  - 需要在多章之间统一抽象层次
  - 需要判断 evidence 挂接是否可靠
  - 需要做最终冻结和跨页一致性裁决
- 成本优先级：
  - 第一优先：`gpt-5.4-mini`
  - 第二优先：只在高风险或最终整合时使用更强模型
