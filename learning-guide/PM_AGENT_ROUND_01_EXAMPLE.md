# 3 PM 子代理首轮示例

本示例面向当前站点的第一轮讨论，目标是帮助你立即启动一次受约束的多 PM 协作，而不是重新定义全站范围。

## 1. 轮次目标

首轮只讨论一个问题：

- 首页首屏是否只保留一个唯一推荐入口，并把证据型入口整体后置

本轮不讨论：

- 全站视觉风格
- 技术实现
- 新增页面
- SEO 和增长

## 2. Shared Brief

### target_user

- 第一次进入学习站的新手或半熟悉用户
- 希望快速建立 Claude Code 主线理解，再进入源码和 analysis 原文

### current_goal

- 把首页从“信息混合面板”改成“明确导学入口”

### page_scope

- `/`

### current_pain_points

- 首页首屏同时出现学习路径、阶段图、源码入口、analysis 入口和补充阅读
- 用户知道站点内容很多，但不一定知道第一步应该点击哪里
- 证据型入口出现过早，容易和导学主线竞争注意力

### current_page_roles

- `/`：导学入口 + 总览 + 证据导航 + 分支推荐
- `/map`：阶段图 + 证据映射 + 分支入口
- `/learn/[slug]`：导学 + 解释 + 证据 + 索引
- `/analysis`：导学 + 原文索引 + 证据跳转
- `/sources`：导学 + 源码索引 + 统计

### must_keep

- 明确的推荐主线
- 从首页进入 `/learn/architecture` 的唯一推荐入口
- 站点具备跳往 `/map`、`/analysis`、`/sources` 的能力

### can_delay

- analysis 原文入口卡片
- sources 全量索引入口
- 首页上的扩展分支推荐

### constraints

- 不修改内容模型
- 不讨论实现方式
- 不新增路由

### last_round_consensus

- 无，首轮

### round_question

- 首页首屏是否只保留一个唯一推荐入口，并把证据型入口整体后置

## 3. 预期输出格式

UI PM 与内容引导 PM 都必须回答：

- 当前问题是什么
- 推荐怎么改
- 明确放弃什么
- 验收标准是什么

整体规划 PM 必须输出：

- 当前共识
- 当前分歧
- 裁决结果
- 本轮唯一结论
- 不做项
- 下一轮唯一问题

## 4. 首轮成功标准

- 用户进入首页 5 秒内能知道唯一推荐入口
- 首页首屏只承担“导学入口”这一项主职责
- 证据型入口仍可访问，但不与首屏主入口竞争
- 下一步跳转清晰，用户知道先去 `/learn/architecture`

## 5. 建议的第二轮候选问题

从以下问题中只选一个进入下一轮：

- `/map` 是否只保留阶段推进和主线路径职责
- `/learn/[slug]` 是否把证据区整体后置
- `/analysis` 是否彻底收敛为证据中心
