# Proposal

## Background

`learning-guide` 已具备真实源码与真实 analysis 驱动的学习底座，但当前产品仍存在明显的导学问题：

- UI 风格偏信息面板化，教程感不足
- 首页、地图页、章节页之间的引导关系不够清楚
- 内容表达形式偏单一，视觉节奏和阅读节奏都较弱
- 页面主次关系不够稳定，用户第一眼不容易判断该从哪里开始

这会直接影响新用户进入路径和学习效率。

## Goal

将 `learning-guide` 优化成更像教程站而不是资料看板的学习产品，重点改善：

- UI 风格统一性
- 学习路径引导清晰度
- 页面主次关系
- 内容展现形式的丰富度与节奏

## Success Criteria

- 首页首屏能明确表达唯一主入口和推荐学习顺序
- 地图页更强调阶段路由，而不是资料平铺
- 章节页形成稳定的“主阅读区 + 辅助侧栏”结构
- 至少新增一类更具教程感的展现形式，用于帮助理解
- 关键页面构建通过，且能在本地运行后通过 `curl` 验证

## Scope

本轮改动聚焦于：

- `/`
- `/map`
- `/learn/[slug]`
- 与上述页面直接相关的展示组件和全局样式

## Non-Goals

本轮不做：

- 内容真实性策略变更
- review / quiz / 学习反馈系统回归
- 真实源码与 analysis 证据层的大规模重构
- 后端、数据持久化或搜索能力改造
