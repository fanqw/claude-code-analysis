# Design

## Decision

本轮采用“教程站化”的改造方向，而不是继续增加新的信息卡片或统计面板。

总体设计原则：

1. 首页负责把用户送上主线
2. 地图页负责阶段选择，不负责证据阅读
3. 章节页负责单章学习，不再像资料汇总页
4. 表达层增加“教程步骤感”和“更轻的视觉语言”

## Information Architecture

### Home

- 强化首屏导语和唯一入口
- 增加“第一次进入该怎么走”的教程式引导
- 让阶段流和推荐顺序成为主内容
- 弱化次级统计和说明性标签

### Map

- 强调阶段路由
- 每个阶段突出“入口章节”
- 将分支阅读与补充路径下沉到次级区域

### Learn Module

- 改成两栏结构
  - 主阅读区：结论、图、解释、伪代码、真实证据入口
  - 辅助侧栏：本章任务、大纲、下一步
- 让用户先读当前章节，再决定跳转

## Presentation Strategy

使用三类更适合教程产品的表达：

- 教程式步骤卡
- 更强的阶段卡和章节入口卡
- 折叠的次级细节区，减少首屏密度

## Styling Direction

- 保留暗色基调
- 压低标题层级和静态标签的视觉强度
- 减少“像按钮但不能点”的视觉元素
- 增强真正可点击入口的清晰度

## Risks

- 页面改得过“教程化”后，可能影响已有证据层入口
- 首页、地图、章节页如果都强调“从这里开始”，会重复
- 章节页两栏布局若控制不好，容易出现右侧过重或移动端退化

## Mitigations

- 将证据入口保留在章节页下层和可展开区
- 首页与地图页分别承担不同职责，避免重复表达
- 所有布局改动都以响应式检查和本地运行结果为准
