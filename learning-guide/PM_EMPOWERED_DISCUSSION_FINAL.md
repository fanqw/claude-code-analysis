# 全责 PM 子代理迭代讨论纪要

## 讨论范围

- 参与角色：3 个全责 PM 子代理
- 讨论中心：`引导用户学习，保证源码真实性`
- 讨论轮次：4 轮
- 讨论目标：收敛网站的核心理念、页面层级、章节模板和导航语义

---

## 第 1 轮：核心理念与母模型

### 共识

- 这个网站不是资料站，也不是文档索引站，而是教程产品
- 网站必须先给用户一条稳定学习主线，再引导用户去做真实性核验
- 任何结论都必须能回到真实源码或真实 analysis 原文
- 页面必须单一职责，不能继续混合导学、正文、证据和分支导航

### 收敛出的母模型

- `教程树 -> 单章正文 -> 证据核验`

### 对应页面语义

- `/` 与 `/map` 属于教程树
- `/learn/[slug]` 属于单章正文
- `/analysis` 与 `/sources` 属于证据核验

### 第 2 轮问题

- `learn/[slug]` 的“正文主线”和“证据核验”边界应该怎么切？

---

## 第 2 轮：章节页边界

### 共识

- 章节页必须固定模板化，不能继续做成内容总汇页
- 正文主线必须先于证据核验
- 用户在章节页内需要完成一个最小闭环，而不是自己判断阅读顺序

### 收敛出的章节页模板

1. `TutorialHeader`
2. `章节导语 / 问题定义`
3. `单章结论`
4. `为什么学这章`
5. `白话解释`
6. `最小主视图`
7. `证据核验区`
8. `扩展层`
9. `Next Step`

### 正文主线必须包含

- 章节标题
- 问题定义
- 单章结论
- 为什么学这章
- 白话解释
- 1 个最小主视图
- 最小下一步

### 证据核验必须包含

- 真实 analysis 原文入口
- 真实源码入口
- 必要的原文或代码摘录
- 对应文件与回跳链接

### 扩展层可包含

- 相关章节
- 先修章节
- 知识点卡片
- 项目映射
- 更多 analysis 视角
- 更深的伪代码

### 最小闭环

- `结论 -> 理解 -> 核验 -> 下一步`

### 第 3 轮问题

- 站点的第一组织原则是否必须是“教程阶段”？

---

## 第 3 轮：导航与页面层级

### 共识

- 第一组织原则必须是“教程阶段”，不能是内容类型或证据类型
- 教程主路径必须先于证据分类
- 首页和地图页不能再被内容分类拖成索引页

### 收敛出的层级关系

- `/`：进入教程树的门口
- `/map`：教程树根目录
- `/learn/[slug]`：教程树上的章节节点
- `/analysis`、`/sources`：挂在章节节点之后的核验分支

### 第 4 轮问题

- `/analysis` 与 `/sources` 是否还需要顶层可见入口？

---

## 第 4 轮：证据中心是否顶层可见

### 多数意见（2/3）

- `/analysis` 和 `/sources` **不保留顶层可见入口**
- 它们只作为章节页中的后置核验入口，外加页脚或章节末尾的辅助入口
- 顶层导航只服务教程阶段，不服务证据分类

### 少数意见（1/3）

- 保留 `证据中心` 这个顶层弱入口
- 但 `/analysis` 和 `/sources` 不再与教程阶段平级，只作为 `证据中心` 下的次级入口

### 最终推荐裁决

采用多数方案：

- 顶层导航不直接暴露 `/analysis` 和 `/sources`
- 顶层导航只保留教程树语义
- 证据核验通过章节页后置区、页脚或直接 URL 访问

裁决理由：

- 用户进入站点时，第一任务是建立主线理解，而不是选择证据类型
- 证据入口作为核验工具更合理，不应再次成为教程主入口的竞争者
- 真实性不会因此丢失，因为每个章节页都要固定保留真实核验入口

---

## 最终冻结方案

### 核心理念

- 网站是教程产品，不是资料站
- 主线学习优先于自由浏览
- 源码真实性必须内置到教程结构里
- 页面职责必须单一

### 最终结构模型

- `教程树 -> 单章正文 -> 证据核验`

### 顶层导航最终建议

- `开始学习`
- `学习地图`
- `阶段 1：系统骨架`
- `阶段 2：长会话治理`
- `阶段 3：安全与扩展`
- `阶段 4：高阶协作`

### 页面角色最终建议

- `/`：唯一教程入口
- `/map`：阶段推进与主线分流
- `/learn/[slug]`：单章正文
- `/analysis`：analysis 原文核验中心
- `/sources`：源码核验中心

### 必须删除或降级的现有组织方式

- 删除所有把 `/analysis`、`/sources` 和主线路径平级并列的顶层入口
- 降级首页和 `/map` 首层里的证据卡片、源码卡片、分析视角卡片
- 删除按内容类型优先组织的第一层导航
- 降级章节页里与正文同级的相关链接、完整映射和扩展索引
- 删除“第二首页”式的 `/map`

---

## 最终实施优先级

1. 顶层导航彻底改成教程阶段优先，移除 `/analysis` 和 `/sources` 的顶层可见入口
2. 章节页统一成固定模板：教程头部 -> 正文主线 -> 证据核验 -> 扩展层
3. 首页和 `/map` 清除所有会与主线竞争的证据与分支入口
4. `/analysis` 与 `/sources` 的文案和结构彻底改成核验中心，不再承担导学职责
5. 做一次全站导航审计，删掉重复入口和语义冲突的链接

---

## 最终验收点

- 首页顶层和首屏都只服务教程主线
- `/map` 只承担阶段推进和分流，不再兼做索引页
- `/learn/[slug]` 的首屏只承担单章正文，不被证据和扩展打断
- `/analysis` 与 `/sources` 只在核验层发挥作用
- 用户可以完整走通 `教程树 -> 单章正文 -> 证据核验`
- 任一关键结论都可回到真实源码或真实 analysis 原文
- 站点不再存在“同一层同时承担教程入口和证据目录”的混合结构
