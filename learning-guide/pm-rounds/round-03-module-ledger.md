# Decision Ledger

## round

- 03

## question

- `/learn/[slug]` 是否应把证据区整体后置，并把章节页收敛为“单章主线 + 证据补充”

## consensus

- `/learn/[slug]` 不能再做成内容总汇页，必须收敛为“单章主线 + 证据补充”
- 首屏的任务只应是帮助用户完成这一章的基本理解
- `真实 evidence` 仍要保留，但只能作为后置补充，不应和主线并列抢注意力

## conflicts

- 分歧不在方向，而在首屏信息量的上限
- UI 方案倾向首屏保留章标题、单章结论、为什么学这章、最小下一步
- 内容方案倾向在主视图里保留最小流程或主视图桥接，但同意其他扩展项后置

## decision

- 定稿为：首屏只保留 `章标题 + 单章结论 + 为什么学这章 + 最小下一步`
- `流程总览`、`analysis` 视角、真实证据、源码映射、项目映射、知识点、相关章节统一下沉到“证据与扩展”区
- “建议阅读顺序”和“先修 / 相关”合并为一个更轻的导航块，避免重复出现多个跳转面板

## non_goals

- 不做首屏并列展示解释、索引、证据和映射
- 不做大量 `related/source` 导航块平铺
- 不做知识点、项目映射、源码映射、analysis 视角抢占首屏
- 不做多层 `details` 继续把主线打散
- 不让章节页继续承担全站信息入口职责

## acceptance_criteria

- 用户能按 `结论 -> 为什么学 -> 怎么理解 -> 去哪里继续` 的顺序读完这一页
- 首屏只出现 1 个单章结论区、1 个为什么学这章区和 1 个最小下一步
- `证据与扩展` 区默认不抢首屏
- 用户不需要滚动很久就能完成第一轮理解
- 移动端首屏不同时承担解释、索引、证据和映射四种职责
