# 学习指引网站统一方案草稿

## 1. 目标与问题定义

- 当前站点失败点：首页首屏同时承担导学入口、总览、证据导航和分支推荐，用户难以快速判断第一步是什么。
- 目标用户：第一次进入学习站的新手或半熟悉用户，需要先建立 Claude Code 主线理解，再进入源码和 analysis 原文。
- 本次改造目标：把首页从信息混合面板收敛成明确的导学入口。

## 2. 用户学习主线

- 用户先看什么：首页唯一推荐入口 `→ /learn/architecture`
- 后看什么：`/map` 看阶段全局 `→` 主线章节 `→` `/analysis` 和 `/sources`
- 为什么这样排：先建立主线，再进入阶段图，最后进入证据中心，避免证据入口和首页导学入口竞争注意力

## 3. 页面职责划分

- `/`：导学入口。只负责回答“先看什么”
- `/map`：暂定为阶段推进和主线分流，等待下一轮确认
- `/learn/[slug]`：暂定为章节导学与解释，等待后续轮次确认证据区位置
- `/architecture`：当前不在本轮裁决范围
- `/analysis`：证据中心，不作为首页首屏主入口
- `/sources`：源码证据中心，不作为首页首屏主入口

## 4. UI 交互原则

- 首屏策略：只保留 1 个强层级 CTA，明确指向 `/learn/architecture`
- 导航策略：`/map`、`/analysis`、`/sources` 保留可达性，但降为次级入口
- 信息密度规则：首屏不再并列展示多个高权重入口
- 展开/折叠规则：证据型入口放入更靠后的“补充阅读 / 证据中心”区块
- 移动端阅读规则：移动端首屏不得出现多于 1 个主要行动按钮

## 5. 内容引导原则

- 内容顺序：先唯一主入口，再辅助理解，再补充入口
- 主线与分支边界：首页优先服务主线，不同时承担分支推荐职责
- 证据何时出现：证据型入口后置到主导学内容之后
- 术语解释规则：首页只保留帮助用户决定第一步的最小必要说明

## 6. 第一批改造项

- 先改哪些页面：先改 `/`
- 每页改什么：
  - `/` 首屏只保留唯一推荐入口
  - `/` 将 `/map`、`/analysis`、`/sources` 降为次级入口
  - `/` 将 analysis 原文入口、sources 全量索引入口、扩展分支推荐整体后置
- 不改什么：
  - 不新增路由
  - 不调整内容模型
  - 不讨论全站视觉风格

## 7. 验收标准

- 页面主职责单一：首页首屏只承担导学入口职责
- 首页唯一主入口清晰：用户 5 秒内能指出 `/learn/architecture`
- 主线可连续走通：用户知道首页之后先去 `/map` 和主线章节，而不是直接跳证据中心
- 首屏信息负担下降：不再出现多个高权重并列入口
- 证据入口不抢引导入口：`/analysis`、`/sources` 不再处于首页首屏强行动区

## 附录

### 轮次记录

- 第 1 轮主题：首页首屏是否只保留一个唯一推荐入口，并把证据型入口整体后置

### 分歧与裁决

- 分歧：次级入口露出的时机和层级
- 裁决：采用“单主入口首屏 + 次级入口下沉 + 证据型入口后置”

### 不做项清单

- 不做首屏多主按钮并列
- 不做 `/analysis`、`/sources` 的首屏强曝光
- 不做首页继续承担完整总览职责
- 不做“入口很多但层级一样”的折中方案
