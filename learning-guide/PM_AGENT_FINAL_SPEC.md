# 学习指引网站最终统一方案

## 1. 目标与问题定义

- 当前站点的核心问题不是内容不够，而是多个页面同时承担导学、总览、证据、索引和分支推荐职责，导致用户很难判断“先看什么”
- 目标用户是第一次进入学习站的新手或半熟悉用户，需要先建立 Claude Code 主线理解，再进入源码和 analysis 证据层
- 本次改造目标是把站点收敛成一条稳定学习主线，并让关键页面只承担单一主职责

## 2. 用户学习主线

- 首页 `/`：只负责把用户送入唯一推荐主线
- 阶段页 `/map`：只负责说明阶段怎么推进、下一步去哪
- 章节页 `/learn/[slug]`：只负责帮助用户完成单章理解
- 证据页 `/analysis` 与 `/sources`：只负责提供原文和源码证据，不再竞争首要学习入口

推荐顺序：

1. `/`
2. `/learn/architecture`
3. `/map`
4. 主线章节
5. `/analysis`
6. `/sources`

## 3. 页面职责划分

### `/`

- 主职责：导学入口
- 只回答：先看什么
- 不再承担：全站总览、证据导航、分支推荐

### `/map`

- 主职责：阶段推进 + 主线分流
- 只回答：阶段怎么走、下一步去哪
- 不再承担：证据中心、源码索引中心、第二首页

### `/learn/[slug]`

- 主职责：单章理解
- 只回答：这一章是什么、为什么学、怎么开始
- 不再承担：内容总汇、全站入口、证据中心

### `/analysis`

- 主职责：analysis 原文证据中心
- 只回答：去哪里看真实 analysis 原文

### `/sources`

- 主职责：源码证据中心
- 只回答：去哪里看真实源码入口和索引

## 4. UI 交互原则

- 首屏只允许一个强层级目标，不并列多个主按钮
- 次级入口必须保留可达性，但不能与主入口平级竞争注意力
- 页面首层只服务单一主职责
- 证据、索引、扩展、分支一律降级到首层之后
- 移动端首屏不得出现超过 1 个主要行动按钮，或超过 2 个同级导航块
- 教程页顶部必须提供稳定路径提示，包括面包屑、阶段标签、主线位置和最小下一步
- `教程正文` 与 `证据核验` 必须分层，正文优先，证据后置

## 5. 内容引导原则

- 先主线，后证据
- 先结论，后展开
- 先最小下一步，后补充分支
- 如果一个内容不能帮助用户进入下一步，默认不放首屏
- 每页只能有一个主职责
- 教程内容默认采用“三层结构”：教程树 → 单章正文 → 证据核验
- 章节页应显示稳定头信息，包括章节目标、阅读时长、所属路径和当前快照时间

## 6. 第一批改造项

### 优先级 P0

- 重构首页 `/`
  - 首屏只保留唯一推荐入口 `/learn/architecture`
  - 把 `/map` 降为次级入口
  - 把 `/analysis` 和 `/sources` 统一后置到“补充阅读 / 证据中心”

- 重构 `/map`
  - 首层只保留阶段推进图、推荐主线、一句页面说明
  - 移除首层证据映射、源码直链、重复伪代码和分支目录

- 重构 `/learn/[slug]`
  - 首屏只保留章标题、单章结论、为什么学这章、最小下一步
  - 把 evidence、analysis 视角、源码映射、项目映射、知识点、相关章节统一后置

### 优先级 P1

- 强化 `/analysis` 与 `/sources` 的证据中心文案，使其不再被理解为首要学习入口
- 清理重复导航块和重复路由链接

## 7. 明确不做项

- 不做首页多主按钮并列
- 不做 `/map` 首屏证据直链和源码入口集合
- 不做章节页首屏并列展示解释、索引、证据和映射
- 不做为了保留所有快捷入口而牺牲首屏清晰度的折中方案
- 不做增长、SEO、社区传播和后端能力讨论

## 8. 验收标准

- 用户 5 秒内能指出首页唯一推荐入口
- `/map` 打开后，用户能用一句话说出“这页告诉我阶段怎么走”
- 章节页首屏能按 `结论 -> 为什么学 -> 怎么开始` 顺序完成第一轮理解
- `/analysis` 与 `/sources` 明确被理解为证据中心，而不是第一学习入口
- 移动端首屏不出现过多同级操作块

## 9. 实施清单

1. 重排首页首屏，移除首屏中的 analysis/source 强入口
2. 精简首页后续区块，只保留主线、辅助理解、补充入口三层
3. 精简 `/map` 首层，删除首层证据映射和分支目录
4. 重构 `/learn/[slug]` 的上半屏，只保留单章主线信息
5. 将 evidence 和扩展区统一下沉并合并重复导航
6. 复查 `/analysis` 和 `/sources` 的首屏文案，确保其定位为证据中心
7. 为教程页补统一头部和稳定路径提示
